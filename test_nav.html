✅ 修复内容：
  • 修复了 handleHashChange() 函数的逻辑
  • 添加了 showSectionWithoutHash() 辅助函数
  • 优化了哈希值处理（移除 '#' 前缀的问题）
  • 确保导航链接点击后正确切换页面
✅ 问题根源：

之前的 handleHashChange() 函数在接收哈希值时，如果不以 #post- 开头，就总是显示首页。这导致点击"关于"、"博客"、"写文章"等链接时，虽然哈希值改变了，但页面仍然回到首页。

✅ 修复方案：
  1. 改进了哈希值的处理逻辑
  2. 添加了三个条件判断：
    • 如果是 post- 开头，显示文章详情页
    • 如果是其他值，显示对应的页面
    • 如果没有哈希值，显示首页
  3. 添加了 showSectionWithoutHash() 来避免循环调用

📝 修改详情

1️⃣ handleHashChange() 函数（第 71-84 行）

原代码问题：

if (hash.startsWith('#post-')) {
    // ...
} else {
    showSection('home');  // 导致问题！
}

新代码：

const hash = window.location.hash.substring(1);  // 移除 '#'
if (hash.startsWith('post-')) {
    // 处理文章页
} else if (hash) {
    showSectionWithoutHash(hash);  // 处理其他页面
} else {
    showSectionWithoutHash('home');  // 默认首页
}

2️⃣ 新增 showSectionWithoutHash() 函数（第 138-162 行）

这个函数与 showSection() 类似，但不会改变 window.location.hash，避免触发无限循环。
